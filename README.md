## ✨ Функціональні можливості

- **⏱ Таймер задач** — запуск/зупинка таймера одним натисканням; тривалість автоматично округлюється до 10 хвилин
- **📋 Журнал активності** — всі дії згруповані по датах із відображенням загального часу за день
- **✏️ Редагування записів** — змінення назви, отримувача, дати та тривалості будь-якої дії
- **↕️ Drag-and-Drop** — сортування завдань усередині дня перетягуванням за маркер `☰`
- **📁 Налаштовувані списки** — управління напрямками, відділами, шаблонами дій та отримувачами безпосередньо в інтерфейсі
- **📊 Експорт звітів** — збереження у форматах Excel (`.xlsx`), PDF та JSON-резервну копію
- **📥 Імпорт даних** — відновлення всього стану застосунку з JSON-файлу
- **📱 PWA** — встановлюється на смартфон або десктоп як нативний застосунок, працює офлайн

## 🚀 Швидкий старт

Застосунок являє собою **один HTML-файл** — жодних залежностей для встановлення не потрібно.

```bash
# 1. Клонуйте репозиторій
git clone https://github.com/your-username/working-time-tracker.git

# 2. Відкрийте файл у браузері
open working-time-tracker-PWA-v2.0.html
```

Або просто перетягніть файл `working-time-tracker-PWA-v2.0.html` у вікно браузера.

> **Для встановлення як PWA** — відкрийте файл через локальний сервер або хостинг, тоді браузер запропонує встановити застосунок.

## 🖥 Як користуватись

### 1. Заповніть дані працівника
Вкажіть напрямок, відділ, посаду та прізвище з іменем у лівій панелі. Ці дані автоматично підставляться в експортовані звіти.

### 2. Оберіть дату та час роботи
Встановіть дату звіту, час початку та завершення робочого дня.

### 3. Почніть відстеження
- Введіть назву функціональної дії (або оберіть шаблон)
- Оберіть, для кого виконується дія
- Натисніть **▶ ПОЧАТИ** — таймер запуститься
- Натисніть **⏹ ЗАВЕРШИТИ** — запис з'явиться в журналі

### 4. Керуйте журналом
| Дія | Як виконати |
|-----|------------|
| Редагувати запис | Кнопка `✎` праворуч від запису |
| Видалити запис | Кнопка `×` праворуч від запису |
| Змінити порядок | Перетягнути за маркер `☰` |
| Згорнути/розгорнути день | Клік по заголовку дня |

### 5. Керуйте списками
Кнопки **Редагувати** / **Шаблони** поруч із кожним полем відкривають менеджер для додавання та видалення елементів списків.

### 6. Експортуйте звіт
| Кнопка | Результат |
|--------|-----------|
| `Excel` | Форматований `.xlsx` файл з даними працівника та всіма задачами |
| `PDF` | Табличний звіт у форматі PDF |
| `↑ JSON` | Резервна копія всіх даних |
| `↓ JSON` | Відновлення з резервної копії |

## 💾 Зберігання даних

Усі дані зберігаються в **LocalStorage** браузера за ключами:
- `tasks_v8.1` — журнал задач
- `lists_v8.1` — налаштовувані списки (напрямки, відділи, шаблони дій, отримувачі)
- `collapsedDays_v8.1` — стан згорнутих груп дат

Для резервного копіювання або переносу між пристроями — скористайтеся кнопками `↑ JSON` / `↓ JSON`.
